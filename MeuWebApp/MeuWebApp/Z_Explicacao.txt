1. Pasta Pages/

A pasta Pages é o núcleo da aplicação. Ela contém todas as páginas Razor, que combinam HTML e C# em um mesmo arquivo, com a extensão .cshtml.
Cada página Razor geralmente possui dois arquivos:

Arquivo .cshtml: contém a estrutura HTML da página e o uso de sintaxe Razor (@{} e @variáveis) para integrar conteúdo dinâmico.

Arquivo .cshtml.cs: contém o código C# de “code-behind”, responsável pela lógica de processamento da página, como leitura de dados, validações e controle de fluxo.

O ASP.NET Core utiliza o modelo PageModel para associar o código da página (.cshtml.cs) ao conteúdo visual (.cshtml).
Quando uma requisição HTTP é feita, o framework localiza a página correspondente dentro da pasta Pages, executa a lógica do arquivo .cshtml.cs e renderiza o HTML final, que é retornado ao navegador.

2. Pasta wwwroot/

A pasta wwwroot é o ponto de acesso público da aplicação. Todo conteúdo armazenado aqui pode ser requisitado diretamente pelo navegador, sem passar pela camada de execução do Razor.
Arquivos comumente encontrados incluem:

CSS (ex.: site.css) — folhas de estilo responsáveis pela aparência visual das páginas.

JavaScript (ex.: site.js) — scripts para comportamento dinâmico no lado do cliente.

Imagens, fontes e ícones.

Essa pasta equivale ao diretório “root” de um site tradicional. Qualquer arquivo nela pode ser acessado via URL direta, por exemplo:
https://localhost:5001/css/site.css.

3. Arquivo appsettings.json

O arquivo appsettings.json é o local onde ficam armazenadas as configurações globais da aplicação em formato JSON.
Ele substitui o antigo web.config das versões anteriores do ASP.NET.
Aqui são definidas informações como:

Strings de conexão com banco de dados;

Configurações de log;

Parâmetros de ambiente e valores personalizados usados pelo código C#.

O ASP.NET Core também suporta arquivos complementares como appsettings.Development.json e appsettings.Production.json, permitindo definir configurações diferentes de acordo com o ambiente em que o sistema está sendo executado.

4. Arquivo Program.cs

O arquivo Program.cs é o ponto de entrada da aplicação. Ele substitui o antigo Startup.cs em versões anteriores do .NET Core.
Dentro dele é criado um objeto WebApplicationBuilder, responsável por:

Configurar o servidor web interno (Kestrel);

Registrar serviços necessários (como Razor Pages, Entity Framework, Identity etc.);

Definir o pipeline de requisições HTTP.

Após a construção da aplicação com builder.Build(), são configurados os middlewares, que controlam o fluxo de cada requisição — por exemplo, redirecionamento HTTPS, tratamento de erros, autorização, entre outros.
Por fim, o comando app.Run() inicia o servidor e mantém a aplicação escutando requisições até ser encerrada.

5. Pasta Properties/

A pasta Properties contém arquivos de configuração específicos do projeto, especialmente o launchSettings.json.
Esse arquivo define como o Visual Studio ou o .NET CLI executa a aplicação, incluindo:

Perfis de execução (por exemplo, IIS Express ou Kestrel);

Portas utilizadas no servidor local;

Variáveis de ambiente de execução;

Configurações de depuração.

Essas definições são utilizadas apenas em tempo de desenvolvimento e não afetam o comportamento em produção.

6. Dependências (Dependencies)

A seção Dependencies lista todas as bibliotecas e pacotes NuGet que o projeto utiliza.
Essas dependências são carregadas automaticamente pelo gerenciador de pacotes do .NET.
Elas incluem tanto componentes internos do ASP.NET Core (como Microsoft.AspNetCore.Mvc.Razor.RuntimeCompilation) quanto pacotes externos adicionados pelo desenvolvedor.

O controle dessas dependências é feito no arquivo *.csproj, que contém as referências e versões dos pacotes utilizados no projeto.
